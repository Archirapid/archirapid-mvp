### ✅ RESUELTO — Intranet Admin: errores PostgreSQL

#### Causa raíz global
`pd.read_sql_query()` con el wrapper `_PostgresConnWrapper` llama `len()` internamente sobre los rows → `_CompatRow` no implementa `__len__` → falla en todos los tabs.

#### Fixes aplicados (commit `67c2863`)
- **`_read_sql9` (tab9 Analytics):** rollback on error + column names en Title Case (PG devuelve alias en minúsculas)
- **`_read3` helper (tab3 Ventas):** reemplazadas 4× `pd.read_sql_query`
- **`_read_tab8` helper (tab8 Actividad):** reemplazadas 3× `pd.read_sql_query`
- **tab7 Waitlist:** reemplazado `pd.read_sql_query` con `conn.execute()` directo
- **tab5 Profesionales:** rollback después de `ALTER TABLE ADD COLUMN` fallido (columnas ya existentes dejaban PG en estado abortado)
- **`src/db.py`:** añadido `rollback()` a `_PostgresConnWrapper`
- **Fix fecha:** `WHERE DATE(created_at) >= DATE('now', '-30 days')` — `adapt_sql` lo traduce a `(created_at)::date >= (CURRENT_DATE + INTERVAL '-30 days')`

#### Waitlist — botones Aprobar/Eliminar (commit `bcc7b0b`)
- Cada entrada pendiente aparece como expander con ✅ Aprobar y 🗑️ Eliminar
- `UPDATE waitlist SET approved=1` / `DELETE FROM waitlist`
