## 1. LOLA — Widget IA chatbot (2-3 abril)

### Problema inicial
Lola era un chatbot IA que debia flotar en la esquina inferior derecha de TODAS las paginas. No funcionaba por multiples razones tecnicas.

### Problemas encontrados y solucionados

#### 1.1 Lola invisible en todas las paginas
- **Causa:** `render_lola()` estaba al final de `app.py` (~linea 2428) pero hay 23+ llamadas `st.stop()` que matan la ejecucion antes de llegar.
- **Solucion:** Mover `from lola_widget import render_lola; render_lola()` a linea ~76 de `app.py`, ANTES del primer `st.stop()`.

#### 1.2 iframe sandbox bloquea teclado
- **Causa:** `st.components.v1.html()` crea iframe con `sandbox="allow-popups allow-scripts allow-popups-to-escape-sandbox"` — SIN `allow-forms`. En algunos navegadores esto bloquea la escritura en `<input>`.
- **Intentos fallidos:** MutationObserver para anadir `allow-forms` (modificar sandbox recarga iframe), `contenteditable` div, `st.markdown` directo.
- **Solucion final:** Tecnica `window.frameElement` — el iframe se auto-posiciona accediendo a su propio elemento padre:
```javascript
var me = window.frameElement;
if (me) {
    me.style.cssText = 'position:fixed!important;bottom:20px!important;right:20px!important;...';
}
```
Esto funciona porque Streamlit no pone `sandbox="allow-same-origin"` pero el iframe SI puede acceder a `window.frameElement` en mismo origen. El input funciona dentro del iframe sin sandbox issues.

#### 1.3 `<script>` no ejecuta en st.markdown
- **Causa:** React `dangerouslySetInnerHTML` NO ejecuta `<script>` tags. Streamlit usa DOMPurify que ademas elimina event handlers (`onerror`, `onclick`).
- **Intentos fallidos:** `<img onerror>` para bootstrap JS (DOMPurify lo elimina), `<textarea>` con JS dentro (se renderiza como texto visible).
- **Solucion:** Abandonar `st.markdown` para JS. Usar `st.components.v1.html()` donde `<script>` ejecuta normalmente dentro del iframe.

#### 1.4 pointer-events:none bloquea clicks
- **Causa:** CSS tenia `pointer-events: none !important` en el iframe con regla `:hover` para reactivar. Catch-22: hover requiere pointer-events activo.
- **Solucion:** Eliminar `pointer-events: none`. El iframe se auto-posiciona y no bloquea nada.

#### 1.5 Groq API no responde
- **Causa:** Originalmente Lola usaba comunicacion parent-iframe via `postMessage`. El padre (Streamlit) no escuchaba.
- **Solucion:** Llamada directa desde el iframe a `https://api.groq.com/openai/v1/chat/completions` via `fetch()`. Sin intermediario Python.

## 2. Panel de Profesionales/Constructores (3 abril)

### Problema 2.1: Login requiere multiples clicks
- **Causa:** `st.success()` antes de `st.rerun()` genera un ciclo de render extra. Estado residual de `login_role` y `_login_show_registro` no se limpiaba.
- **Solucion:** Eliminar `st.success()` antes de `st.rerun()`. Limpiar `login_role` y `_login_show_registro` con `st.session_state.pop()` tras login exitoso.
- **Archivo:** `app.py` lineas ~1922-1944

### Problema 2.2: Sin boton "Cerrar sesion"
- **Causa:** No existia.
- **Solucion:** Boton `st.button("Cerrar sesion")` debajo del header del panel. Limpia: `logged_in`, `user_id`, `user_email`, `role`, `user_name`, `viewing_login`, `login_role`, `selected_page`. Despues `st.query_params.clear()` + `st.rerun()`.
- **Archivo:** `modules/marketplace/service_providers.py` linea ~444

### Problema 2.3: Badge "DESTACADO" sin sincronizar con admin
- **Causa:** `_is_featured()` fue modificada incorrectamente para aceptar `featured_plan='destacado'` como condicion suficiente. Un proveedor que SELECCIONO el plan de pago al registrarse veia "DESTACADO" aunque admin no lo hubiera activado.
- **Solucion:** Revertir `_is_featured()` a solo verificar `is_featured=1` (flag que ponen admin o Stripe webhook). Ahora hay 3 estados de banner:
  1. `is_featured=1` → "PLAN DESTACADO ACTIVO"
  2. `featured_plan='destacado'` pero `is_featured=0` → "Pendiente de activacion"
  3. Plan gratuito → "Plan Gratuito — X/3 ofertas restantes"
- **Archivo:** `modules/marketplace/service_providers.py` lineas 57-76 y 386-420

### Problema 2.4: Error PostgreSQL `UndefinedFunction: datetime()`
- **Causa:** Query SQL usaba `datetime('now','-24 hours')` — funcion SQLite que no existe en PostgreSQL.
- **Solucion:** Calcular cutoff en Python: `cutoff_24h = (datetime.utcnow() - timedelta(hours=24)).strftime(...)` y usar placeholder `?` en la query.
- **Archivo:** `modules/marketplace/service_providers.py` linea ~494

## 3. Panel Admin/Intranet — Gestion de Profesionales (3 abril)

### Problema 3.1: Sin boton "Acceder" en login admin
- **Causa:** Solo tenia `st.text_input` que validaba al pulsar Enter. Sin boton explicito.

- **Archivo:** `modules/marketplace/intranet.py` lineas 42-55

### Problema 3.2: Sin controles de aprobacion/bloqueo/eliminacion
- **Causa:** Solo existia checkbox Destacado + "Guardar plan" sin explicacion, ademas de que el boton "Guardar plan" no tenia descripcion de su funcion.
- **Solucion:** Panel completo reconstruido. Cada proveedor ahora tiene:

  - **Aprobar/Bloquear:** Si activo → boton "Bloquear" (pone `active=0` + quita `is_featured`). Si bloqueado → boton "Aprobar" (pone `active=1`).
  - **Eliminar:** Boton "Eliminar" con confirmacion doble ("Si, eliminar" / "Cancelar"). Borra de `service_providers` y `users`.
  - **KPI nuevo:** Metrica "Bloqueados" en dashboard
- **Archivo:** `modules/marketplace/intranet.py` lineas 535-650

### Problema 3.3: Proveedor bloqueado podia acceder al panel
- **Causa:** `show_service_provider_panel()` no verificaba `active`.
- **Solucion:** La SELECT ahora incluye `active`. Si `active=0`, muestra error "Tu cuenta ha sido desactivada" y hace `return`.
- **Archivo:** `modules/marketplace/service_providers.py` lineas 354-376

- **Como funciona:** Admin marca checkbox "Activar Destacado" + Guardar. Esto pone `is_featured=1` directamente en la BD sin pasar por Stripe. El proveedor ve exactamente lo mismo que uno que pago. No hay distincion visible.

## Lecciones tecnicas criticas (para futuras sesiones)

### Streamlit + JavaScript
1. **`st.markdown(unsafe_allow_html=True)`** pasa por DOMPurify → ELIMINA `<script>`, `onerror`, `onclick` y todos los event handlers. NO se puede ejecutar JS.
2. **`st.components.v1.html()`** crea iframe con sandbox limitado. `<script>` SI ejecuta dentro del iframe.
3. **`window.frameElement`** permite al iframe acceder a su propio elemento en el DOM padre y auto-posicionarse con CSS. Funciona en mismo origen.
4. **Nunca poner `pointer-events: none`** en un iframe flotante — bloquea todo, incluyendo `:hover`.

### Streamlit session_state
5. **Limpiar estado residual** tras login: `st.session_state.pop('login_role', None)` evita que reruns futuros interfieran.
6. **No poner `st.success()` antes de `st.rerun()`** — genera ciclo de render extra que causa multi-click en formularios.
7. **`st.stop()`** mata la ejecucion — cualquier widget que deba aparecer en TODAS las paginas va ANTES del primer `st.stop()`.

### PostgreSQL vs SQLite
8. **`datetime('now','-24 hours')`** es funcion SQLite — NO existe en PostgreSQL. Calcular cutoff en Python.
9. **Placeholders:** `?` en SQLite, `%s` en PostgreSQL. El wrapper `db_compat.py` los traduce.

### Admin/Destacado
10. **`is_featured` flag** es la unica fuente de verdad para status Destacado. Lo pone admin (intranet) o Stripe (webhook/callback). `featured_plan` solo registra lo que el usuario eligio al registrarse — NO determina acceso.
